# Sprint 7 게임 비주얼 방향

## 목표

기존 매치-3 규칙과 저장 구조를 그대로 유지하면서 게임 화면을 `Berry Gift Pop` 콘셉트로 재구성한다. 플레이 중에는 앱 셸보다 보드, 이동 수, 점수, 미션을 우선하며, 모든 사용자 노출 문구는 한글 프레젠테이션 레이어를 거친다.

## 화면 구조

- 게임 라우트의 플레이 상태는 immersive mode를 사용한다. 일반 상단 바와 하단 탭은 숨기고, 뒤로가기·스테이지 번호·게임 메뉴만 둔다.
- HUD는 남은 이동, 점수, 최대 2개 미션을 한 화면에 표시한다. 남은 이동이 5 이하가 되는 순간 한 번만 주의를 준다.
- Phaser 캔버스는 중앙의 디저트 트레이 프레임 안에 배치한다. 실제 게임 조작은 캔버스만 담당하고 HUD와 결과 화면은 React가 담당한다.
- 결과 화면은 승리와 실패를 분리하고, 실제로 동작하는 다음 스테이지·다시 하기·스테이지 선택·홈 이동만 노출한다.

## 베리 프레젠테이션

내부 `TileKind`는 변경하지 않는다. 사용자 화면에서는 다음 고정 매핑을 사용한다.

| 내부 값  | 사용자 이름 | 텍스처              |
| -------- | ----------- | ------------------- |
| `heart`  | 딸기        | `berry-strawberry`  |
| `star`   | 체리        | `berry-cherry`      |
| `flower` | 블루베리    | `berry-blueberry`   |
| `candy`  | 청포도      | `berry-green-grape` |
| `gift`   | 라즈베리    | `berry-raspberry`   |

각 SVG는 투명한 128×128 아트보드, 외곽선, 하이라이트, 서로 다른 실루엣을 사용한다. BootScene에서 한 번만 로드하고 TileView는 Sprite로 표시한다. 선택 상태는 확대·광택 링·짧은 펄스로 표현하며 문자를 덧씌우지 않는다.

## 특수 타일과 이펙트

- 가로/세로 특수 타일은 방향을 즉시 구분할 수 있는 빛띠를 사용한다.
- 폭탄은 중앙 코어와 방사형 강조, 프리즘은 다색 링과 반짝임을 사용한다.
- 일반 팝은 타일 색을 따른 짧은 파티클, 특수 팝은 더 큰 링과 스파클을 사용한다.
- 콤보는 `N콤보`로 표시하며 2~~3, 4~~6, 7~9, 10+ 단계별 크기·색·파티클 강도를 높인다. 10+는 `하임팡!` 시그니처 버스트를 함께 사용한다.
- 모든 연출은 660ms 이내에 정리되며 게임 상태와 입력 상태를 변경하지 않는다.

## 디자인 토큰

색상, 모서리, 그림자, 간격, 모션은 `tokens.css`와 `designTokens.ts`에 집중한다. 사용자 화면은 로컬 시스템 한글 폰트 스택을 사용하고 외부 폰트 네트워크 요청에 의존하지 않는다.

## 개발 전용 QA

`settings.debugPanel === true`이며 개발 빌드일 때만 Design Lab을 노출한다. 베리 5종, 특수 타일 4종, 콤보 단계, 승패 결과, HUD 상태를 한 화면에서 비교할 수 있다. Design Lab과 Effect Lab은 저장 데이터, 점수, 이동 수, 미션, 보드를 변경하지 않는다.
